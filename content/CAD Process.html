下載隨身系統並建立 Github 倉儲

每一位學員下載 2019Fall可攜套件.7z (編譯  Solvespace 用), 存放於採  extFAT 格式, 支援 USB 3.0 以上連線速度的 USB 隨身碟中, 在 Windows 10 64 位元電腦中, 以 64 位元 Portablegit 擷取存放於 Github 上的個人倉儲資料.

在 Github 倉儲中的畫面如下:

Git Clone 倉儲內容至隨身碟

使用者在近端利用隨身系統中的 git 指令, 將 Github 上的倉儲 clone 到近端隨身碟中:

git clone https://github.com/使用者名稱/倉儲名稱.git

倉儲資料將會以"倉儲名稱"作為目錄名, 將倉儲資料存入.

接著使用者利用 git submodule add 指令, 將 https://github.com/mdecourse/cmsimde.git 倉儲設為子模組, 且存入 cmsimde 目錄中.

git submodule add https://github.com/mdecourse/cmsimde.git cmsimde

完成後, 使用者必須將 cmsimde 目錄中 up_dir 目錄內容複製到前述的"倉儲名稱"目錄內. 而此目錄中包含 CMSiMDE 網際內容管理系統可讓使用者設定的 init.py, config/config, local_publishconf.py, pelicanconf.py 以及 publishconf.py 等檔案. 其中 init.py 與 config/config 設定 CMSiMDE 動態與靜態網站系統所使用的 html 頁面標題與內容管理系統標題. 而 local_publishconf.py, pelicanconf.py 以及 publishconf.py 則用來設定 Pelican 近端與遠端網誌系統內容.

修改倉儲中 .git/config 中的  url 設定

從 https 連線協定改為以 ssh 方式連線.

Github SSH 連線設定

利用 batch 檔案啟動網站內容編輯

接下來使用者可以利用下列 batch 檔案執行, 啟動近端動態網站系統, 並帶出近端動態網站編輯器.

@echo off
y:
cd tmp\cad2019\cmsimde
start /WAIT python wsgi.py | sleep 2 | start chrome https://localhost:9443
cd \
cls

根據第一階至第三階網頁標題方式, 在近端動態系統中編輯內容, 使用者可以利用 HTML語法將圖片, 影片, 檔案下載與程式碼內容放入網頁中.

完成動態系統內容編輯後, 利用 generate_pages 按鈕將 config/content.htm 轉為 content 目錄中靜態網站中的各分頁內容檔案, 此時在動態系統中的各頁面標題, 將作為 content 目錄中各檔案存檔的檔名依據.

將近端倉儲改版內容新增提交推送至  Github

使用者完成近端倉儲改版後, 可以利用下列 batch 檔案, 檢查轉出的靜態檔案內容是否正確:

@echo off
y:
cd tmp\cad2019\
start python http-server.py | sleep 2 | start chrome https://localhost:8444
cd \
cls
